Usage: retest [options] [COMMAND]

Watch files and rerun matching tests when they change.

Arguments:
    COMMAND                          Command to rerun when a file changes.
                                     Use <test> for the matching test file and
                                     <changed> for the changed file.

Options:
        --all                        Run the full test suite for the selected setup
        --diff BRANCH                Run tests matching files changed from BRANCH
        --exts EXTENSIONS            Comma-separated file extensions to watch (default: rb)
    -h, --help                       Show this help
        --notify                     Play a sound when tests pass or fail (macOS only)
        --polling                    Use polling for file watching
        --rails                      Use the standard Rails test command
        --rake                       Use the standard Rake test command
        --rspec                      Use the standard RSpec test command
        --ruby                       Use the standard Ruby test command
    -v, --version                    Show retest version
    -w, --watcher WATCHER            File watcher to use (listen or watchexec)

Examples:
    $ retest
        Auto-detect the project setup and rerun matching tests

    $ retest --all
        Auto-detect the project setup and run the full suite

    $ retest --rails
        Use the standard Rails test command

    $ retest 'bin/rails test <test>'
        Rerun the matching Rails test file

    $ retest 'rubocop <changed> && bin/rails test <test>'
        Run a check on the changed file, then run the matching test

    $ retest --diff main
        Run tests matching files changed from main
